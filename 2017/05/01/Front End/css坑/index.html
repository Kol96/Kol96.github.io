margin折叠

参考
折叠的解决办法:

  • margin折叠元素只发生在块元素上
  • 将父元素与它的第一子元素「隔离开」，比如给父元素加 padding 或 border
  • 为父元素创建块级格式化上下文（BFC），让它的子元素的 margin 值不影响父元素的 margin 值计算关于body
  • 浮动元素不与其他元素margin折叠
  • 使用before伪类给父元素添加一个空元素:before{content:'';display:table;}

1、元素的上外边距和其属于常规流中的第一个孩子的上外边距
一个盒子如果没有padding-top、border-top、float、overflow等属性时，那么这个盒子的上边距会和其内部文档流中的第一个子元素的上边距重叠。当发生重叠时，是按照谁上边距大就取谁作为这个盒子的上边距。

上面的例子我们在浏览器中会看到box元素与general元素之间的空隙应为20px，因为top子元素的上边距大于box父元素的上边距。如果给box父元素增加padding-top、border-top、float、overflow:hidden任一属性，都会让margin-top重叠失效，结果会是box与general元素的空隙变为10px，top元素的margin会施加在box父元素的内部。

2、元素的下外边距和其属于常规流中的最后一个孩子的下外边距且父亲的高度计算值为auto

重点是只有box父元素的高度为auto，即为设置时才会产生这种情况的折叠。

3、元素的下外边距和其属于常规流中的下一个兄弟的上外边距

两个general元素之间的间隙为50px，可以通过给第二个元素加float:left/right或display:inline-block取消margin折叠。

ie6双边距bug

一个块级元素同时设置了向左浮动float:left及左边距margin-left后，则该元素的左边距或右边距在某些情况下会是设定值的两倍。同样地，向右浮动float:right及右边距margin-right也存在此现象。

解决办法:

  • 尽量避免同时使用margin-left与float:left及margin-right与float:right
  • 通过设置 display:inline
