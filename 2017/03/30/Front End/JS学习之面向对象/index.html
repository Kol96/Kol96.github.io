JavaScript学习之面向对象

Front End
  • JavaScript

ECMA-262把对象定义为：无序属性的集合，其属性可以包含基本值、对象或者函数。对象可以看成一组没有特定顺序的值，对象的每个属性或方法都有一个名字，而每个名字都映射到一个值。

三、工厂模式

解决:使用同一个接口创建很多对象，产生大量重复代码。
缺点:无法知道对象的类型。每个实例包含的同名方法不共享。

四、构造函数模式

解决:识别对象类型
缺点:每个实例包含的同名方法不共享。

new关键字:

  • 声明一个中间对象。
  • 将该中间对象的原型指向构造函数的原型。
  • 将构造函数的this，指向该中间对象。
  • 返回该中间对象，即返回实例对象。

五、原型模式

解决:实例共享原型对象的属性和方法
缺点:引用类型的值被所有实例共享

1、创建自定义构造函数后，其原型对象会默认有一个constructor属性。调用构造函数后创建的新实例有个内部属性[[prototype]]指向构造函数的对象。
2、代码读取某个对象的某个属性时，先搜索对象实例本身，如果没找到会搜索指针指向的原型对象，如果没找到会按照原型链找下去。
3、用hasOwnProperty()方法判断访问的是否是实例属性还是原型属性。

4、用in判断访问的是否是当前对象的属性

六、组合使用构造函数模式和原型模式

构造函数模式用于定义实例属性，原型模式用于定义方法和共享属性。
