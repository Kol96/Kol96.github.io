Java随笔之类与对象

一、构造方法

方法名与类名相同 且不能有返回值 用户不能直接调用 通过new新建实例 类名 对象名=new 类名（） 用途：对成员变量进行初始化 无参构造方法只可以第一句使用this调用有参构造方法

二、方法重载

一个类有多个相同名字的方法，但参数类型或数量不同。构造方法也可重载，重载不能通过返回值区分
静态方法中 不可以使用this关键字 不可以直接调用非静态方法

三、封装

public：全都可以调用
protected：包中其他类和子类、本类
private：本类
无修饰符：包中其他类、本类

四、继承

方法重写导致父类被覆盖 想访问父类 需要用super
super.成员变量 super.成员方法 super([参数列表])—构造方法

五、父类子类的类型转换

显式转换：（类名）对象名
子类变父类 可以用显式或隐式转换
parent p=new child()—隐式
父类变子类只能显式
parent p=new parent()
child b=(child)p

六、包

package pk1.pk2.pk3
import 包名.(类名|*)

七、接口

[修饰符]interface 接口名｛｝
接口中所有的方法都是抽象的（abstract可以缺省）
所有数据都是静态常量（public，static，final可以缺省）
class 类名 implements 接口名，接口名，接口名[extends 类名]｛｝
方法需要重写

八、内部类与匿名类

成员内部类（成员位置）
局部内部类（方法位置）
静态内部类（类似静态成员）
匿名内部类（无名，声明和创建合二为一，new <类或接口> <类的主体>
